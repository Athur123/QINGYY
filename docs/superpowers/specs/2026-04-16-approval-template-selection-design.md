# 审批模板选择功能设计

**日期：** 2026-04-16
**状态：** 已确认

---

## 1. 概述

在"申请垫付"页面增加审批模板选择功能，用户可选择已同步至企业微信的垫付申请审批模板，提交时使用该模板发起审批流程。

---

## 2. UI 变更

### 2.1 布局位置

页面顶部、"基础信息"区块上方，新增独立的审批模板卡片。

### 2.2 布局结构

```
┌─────────────────────────────────────────────────────────┐
│  ⚠ 请选择审批模板（必填）                                   │  ← 顶部提示条（红色背景）
├─────────────────────────────────────────────────────────┤
│  * 审批模板                                              │
│  [ 垫付申请模板v2 ▼                              ]      │  ← 下拉选择
│  已同步至企业微信 | 版本：v2 | 最后更新：2026-04-10        │  ← 辅助信息
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┤
│ 基础信息                                                 │
│  客户名称    [新福通2025                         ]      │
│  合同       [新福通BUFF2025/XFT20250001        ]      │
│  ...                                                  │
└─────────────────────────────────────────────────────────┘
```

### 2.3 组件规格

| 元素 | 规格 |
|-----|------|
| 顶部提示条 | 背景 `#FEF2F2`，文字 `#DC2626`，左侧警告图标 |
| 标签 | `* 审批模板`，红色星号 |
| 下拉框 | 与现有下拉样式一致，宽度100% |
| 辅助信息 | 灰色小字 `#64748B`，显示模板状态 |

---

## 3. 数据流

```
用户选择模板 → 表单保存模板ID → 提交审批时传入模板ID → 企业微信创建审批实例
```

### 3.1 前端表单字段

```json
{
  "approval_template_id": "选中模板的ID"
}
```

---

## 4. API 设计

### 4.1 获取模板列表

```
GET /api/approval/templates?type=advance_apply&platform=wecom&sync_status=synced
```

**响应：**
```json
{
  "code": 0,
  "data": [
    {
      "id": 1,
      "name": "垫付申请模板v1",
      "wecom_template_id": "xxxx",
      "last_sync_time": "2026-04-10T10:00:00Z"
    },
    {
      "id": 2,
      "name": "垫付申请模板v2",
      "wecom_template_id": "yyyy",
      "last_sync_time": "2026-04-15T14:30:00Z"
    }
  ]
}
```

### 4.2 提交审批

提交时将 `approval_template_id` 传入后端，后端根据 `wecom_template_id` 调用企业微信 `create_approval` 接口。

---

## 5. 交互逻辑

| 场景 | 行为 |
|-----|------|
| 未选择模板点击"保存并提交" | 顶部提示条高亮闪烁，聚焦下拉框 |
| 未选择模板点击"保存为草稿" | 允许保存，不校验模板 |
| 模板下拉为空 | 显示"暂无可用模板，请联系管理员配置" |
| 选中模板后 | 辅助信息显示模板版本和同步时间 |

---

## 6. 依赖

- 后端需实现 `GET /api/approval/templates` 接口，支持 `type`、`platform`、`sync_status` 参数过滤
- 模板数据来源：`approval_template` 表，`type=advance_apply`，`platform=wecom`，`sync_status=synced`
