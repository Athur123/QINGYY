# 全局字段配置：自定义分类（多选）设计

## 背景与目标
该页面仅承担“全局字段管理（字段字典/字段资产）”职责：创建并维护可被其他业务模块引用的字段。业务模块在各自页面中选择一个或多个字段进行使用，不在本页配置具体业务规则。

本次改动目标（快速配置优先）：
- 支持“自定义分类”，用于快速检索和理解字段资产
- 支持“字段绑定多个分类”（多选）
- 分类维护仅在“字段新增/编辑抽屉”内完成（可选择、可创建），不新增单独分类管理页
- 列表增加“分类”展示列，并提供“分类筛选”

非目标：
- 不引入审批/发布/版本/审计等治理能力
- 不在本页配置“业务模块如何使用字段”的逻辑
- 不做复杂的分类治理（合并/重命名/停用等）

## 用户与使用场景
- 字段管理员：快速创建字段，并用分类帮助后续在业务侧挑选字段
- 业务配置人员：通过字段名称/编码/分类，在业务模块中快速定位字段（本页仅提供“更易检索/更易理解”的字段定义）

## 信息架构与页面改动
### 1) 数据模型（原型内）
新增分类字典 `categories`（示例结构）：
- `id: number`
- `name: string`

字段结构新增：
- `categoryIds: number[]`（可为空；多选）

分类命名规则（轻量）：
- `trim` 前后空格
- 非空
- 同名去重（不区分大小写 + 忽略多余空白的规范化匹配）
- 建议限制长度（例如 2–12），超出提示

### 2) 列表区
- 新增一列：分类
  - 展示形式：tag
  - 最多展示 2 个，超过显示 `+N`
  - 鼠标悬停可显示完整分类列表（tooltip 或 title）
- 筛选区新增：分类筛选（可搜索下拉）
  - 支持选择“全部分类/某一分类”
  - 与现有的关键字、类型、来源筛选组合生效

### 3) 新增/编辑字段抽屉
- 新增字段项：分类（多选）
  - 支持搜索选择已有分类
  - 支持输入创建：当输入内容不匹配已有分类时，提供“创建分类：xxx”，创建后自动选中
  - 分类为非必填：允许不选择分类直接保存

交互与可用性：
- 新建分类成功：toast 提示（可选）
- 重复分类名：不重复创建，直接选中已有分类

## 交互细节
### 列表 tag 显示规则
- `categoryIds.length === 0`：显示 `-` 或 “未分类”
- `1-2`：显示对应 1-2 个 tag
- `>=3`：显示前 2 个 tag + `+${n-2}`

### 筛选逻辑
- 关键字：匹配字段名称/编码
- 类型/来源：按现有逻辑
- 分类：字段的 `categoryIds` 包含所选分类 id

## 校验与错误提示
- 分类输入：空值不允许创建（提示“请输入分类名称”）
- 分类同名：提示“分类已存在”，并自动选中既有分类
- 字段保存：分类本身不参与必填校验（保持快速配置）

## 验收标准
- 列表新增“分类”列，字段可展示多个分类 tag
- 筛选区新增分类筛选，能与现有筛选组合使用
- 抽屉支持分类多选，并支持直接创建分类
- 分类非必填：不选分类也能成功保存字段
