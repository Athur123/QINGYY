# 社保对账复核功能设计

## 1. 概述

### 1.1 背景与问题

当前HRO系统中，社保公积金费用测算基于客户提交的增减员名单进行预收，无论增减员是否申报成功均会纳入测算。这导致系统测算数据与社保局实际台账数据存在差异，包括：

- 增减员申报成功与失败的状态不同步
- 客户反复提交相同名单导致重复测算
- 增减员申报成功后，实际执行日期与预计日期不一致

人力资源公司需要定期核对系统应缴明细与社保局台账明细，定位差异、避免人员申报遗漏。

### 1.2 产品定位

**对账复核系统** — 面向内部HR运营人员，用于核对系统测算数据与社保局台账数据的差异，核心目的是发现人员申报遗漏或错误。

### 1.3 核心场景

```
系统记录（内部） ←——比对——→ 社保局台账（外部导入Excel）
     ↓                                        ↓
 "我以为该缴的人"                    "实际真正缴了的人"
```

## 2. 功能架构

### 2.1 页面入口

入口位置：现有「员工参保档案」页面工具栏区域，新增「对账复核」按钮。

│  │ 应缴 1,256人  │  │ 实缴 1,241人  │  │ 差异 15人 (1.2%)      │    │

### 2.3 三个视图

| 视图 | 显示内容 | 用途 |
|-----|---------|-----|
| **差异明细**（默认） | 只显示两边不一致的记录 | 快速定位问题 |
| **系统全量** | 显示筛选范围内所有系统记录 | 核对台账缺失部分 |
| **台账全量** | 显示导入的所有社保局记录 | 核对系统遗漏部分 |

## 3. 数据模型

### 3.1 匹配粒度

匹配单位：**员工 × 险种 × 费用月份**

同一员工同一月份可能有多条差异记录（按险种拆分）。

### 3.2 系统记录字段

| 字段 | 说明 |
|-----|-----|
| 姓名 | 员工姓名 |
| 身份证号 | 唯一标识 |
| 客户 | 归属的客户公司 |
| 结算主体 | 结算主体 |
| 参保方案 | 社保方案名称 |
| 险种 | 养老/医疗/失业/工伤/生育/公积金 |
| 申报状态 | 待申报/申报中/申报成功/申报失败 |
| 申报时间 | 实际申报日期 |
| 费用月份 | YYYY-MM |

### 3.3 台账记录字段（导入）

| 字段 | 是否必填 | 说明 |
|-----|---------|-----|
| 姓名 | 是 | 员工姓名 |
| 身份证号 | 是 | 唯一标识 |
| 险种 | 是 | 养老/医疗/失业/工伤/生育/公积金 |
| 费用月份 | 是 | YYYY-MM格式 |

### 3.4 差异类型定义

| 差异类型 | 判定规则 | 业务含义 |
|---------|---------|---------|
| **系统多** | 系统有该记录，台账无 | 可能漏申报、申报失败未通知、不该减员被减了 |
| **台账多** | 台账有该记录，系统无 | 可能漏采集名单、误增员 |

## 4. 交互设计

### 4.1 差异明细表字段

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ 费用月份 │ 姓名 │ 身份证 │ 客户 │ 险种 │ 系统状态 │ 差异类型 │ 操作       │
└─────────────────────────────────────────────────────────────────────────────┘
```

**说明：**
- 系统状态 = 该条记录在系统中的申报状态
- 差异类型：系统多 / 台账多

### 4.2 差异详情弹窗

触发方式：点击差异明细表中任意一条的「查看详情」

```
┌─────────────────────────────────────────────┐
│  差异详情：张三 - 2026年4月 - 养老保险         │
├─────────────────────────────────────────────┤
│  系统记录              │  社保局台账           │
│  ─────────────────────│─────────────────────  │
│  客户：某科技有限公司    │  （台账不显示客户）   │
│  参保方案：长沙标准方案  │                      │
│  申报状态：申报成功     │  （台账无此字段）     │
│  申报时间：2026-04-05  │                      │
├─────────────────────────────────────────────┤
│  差异类型：系统多                              │
│  可能原因：申报失败未通知 / 不该减员被减了       │
├─────────────────────────────────────────────┤
│  [标记已核查]  [调整系统记录]  [关闭]           │
└─────────────────────────────────────────────┘
```

### 4.3 操作按钮

| 按钮 | 功能 |
|-----|-----|
| 导入台账Excel | 打开文件选择器，上传社保局导出的Excel |
| 导出差异明细 | 导出当前筛选条件下的差异记录为Excel |
| 查看系统全量 | 切换到系统全量Tab |
| 查看台账全量 | 切换到台账全量Tab |
| 查看详情 | 打开差异详情弹窗 |
| 标记已核查 | 将该条差异标记为已确认，从默认视图移除 |
| 调整系统记录 | （可选）跳转至该员工参保信息页 |

### 4.4 筛选条件

| 维度 | 类型 | 说明 |
|-----|-----|-----|
| 费用月份 | 下拉 | 必选，默认当前月份 |
| 参保主体 | 下拉 | 按地区筛选 |
| 结算主体 | 下拉 | 按结算主体筛选 |
| 方案 | 下拉 | 按社保方案筛选 |
| 客户 | 搜索框 | 支持模糊搜索 |
| 险种 | 下拉 | 养老/医疗/失业/工伤/生育/公积金/全部 |

## 5. Excel导入要求

### 5.1 台账文件格式

| 字段 | 是否必填 | 说明 |
|-----|---------|-----|
| 姓名 | 是 | 员工姓名 |
| 身份证号 | 是 | 唯一标识 |
| 险种 | 是 | 养老/医疗/失业/工伤/生育/公积金 |
| 费用月份 | 是 | YYYY-MM格式 |

### 5.2 匹配逻辑

- 以 `身份证号 + 险种 + 费用月份` 作为唯一键
- 姓名作为辅助校验（身份证相同但姓名不同时报警）

### 5.3 导入限制

- 每次导入覆盖该费用月份该参保主体的已有台账数据
- 不允许跨费用月份混合导入

---

## 6. 边界情况处理

| 场景 | 处理方式 |
|-----|---------|
| 同一员工同一险种台账有多条记录 | 提示用户去重，只保留一条 |
| 身份证号格式不规范 | 导入时提示具体行号和错误原因 |
| 系统无该客户的台账数据 | 台账全量视图显示为空，不报错 |
| 同一员工同一险种系统有多条记录 | 取费用月份最晚的一条 |

---

## 7. 非功能性要求

| 要求 | 说明 |
|-----|-----|
| 性能 | 单次导入控制在5分钟内完成，差异比对在3秒内响应 |
| 容量 | 单月单参保主体最大支持10万条记录 |
| 数据保留 | 台账数据保留24个月 |
| 权限 | 仅管理员和HR运营角色可访问 |
