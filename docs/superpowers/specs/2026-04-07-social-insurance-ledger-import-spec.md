# 社保台账导入规范

## 一、文件规格

### 1.1 格式要求

| 项目 | 要求 |
|-----|-----|
| 文件格式 | Excel (.xlsx / .xls) 或 CSV |
| 编码 | UTF-8 |
| 单文件上限 | 10万行 |
| 费用月份 | 单次导入仅限单一月份 |

### 1.2 字段标准

| 序号 | 字段名 | 必填 | 格式 | 说明 |
|:---:|-------|:---:|------|-----|
| 1 | 姓名 | ✓ | 文本，不超过50字符 | 员工真实姓名 |
| 2 | 身份证号 | ✓ | 18位纯数字或15位 | 唯一身份标识，用于记录匹配 |
| 3 | 险种 | ✓ | 固定枚举值（见下表） | 参保险种类型 |
| 4 | 费用月份 | ✓ | YYYY-MM | 社保费用所属月份 |
| 5 | 缴费基数 | ✓ | 数字，最多2位小数 | 社保申报基数 |
| 6 | 单位缴费额 | ✓ | 数字，最多2位小数 | 当月单位实缴金额 |
| 7 | 个人缴费额 | ✓ | 数字，最多2位小数 | 当月个人实缴金额 |

**险种枚举值（必须严格按以下任一填写）：**

| 枚举中文名 | 英文标识（可选） |
|-----------|----------------|
| 养老保险 | pension |
| 医疗保险 | medical |
| 失业保险 | unemployment |
| 工伤保险 | work_injury |
| 生育保险 | maternity |
| 住房公积金 | housing_fund |

### 1.3 选填字段

| 字段名 | 格式 | 说明 |
|-------|------|-----|
| 参保状态 | 正常 / 暂停 / 注销 | 当月参保状态 |
| 申报到账月份 | YYYY-MM | 实际申报到账月份 |

## 三、导入校验规则

### 3.1 前置校验（阻断型，发现后不允许导入）

| 校验项 | 错误提示 | 处理方式 |
|-------|---------|---------|
| 身份证号格式错误 | 第N行：身份证号格式不正确，应为18位或15位数字 | 修正后重新导入 |
| 必填字段为空 | 第N行：[字段名] 为必填项，不能为空 |  |
| 险种枚举值错误 | 第N行：险种"XXX"不在标准枚举列表中 |  |
| 费用月份格式错误 | 第N行：费用月份格式不正确，应为YYYY-MM |  |
| 数字字段含非数字字符 | 第N行：[字段名] 应为数字格式 |  |

### 3.2 匹配预警（非阻断，提醒人工关注）

| 预警类型 | 提示信息 | 处理方式 |
|---------|---------|---------|
| 身份证号重复 | 第N行与第M行：身份证号相同但姓名不同，请核实 | 人工确认后选择保留哪条 |
| 姓名相似度低 | 身份证号"XXX"对应姓名与系统记录差异较大，请核实 | 人工确认是否同一人 |
| 金额为零 | 第N行：[险种] 缴费金额为0，请确认是否正常 | 人工确认 |

### 3.3 导入后提示

| 提示类型 | 内容 |
|---------|-----|
| 导入成功 | 共导入 N 条记录，其中系统匹配成功 M 条，差异 N-M 条 |
| 覆盖提示 | 该月份已有台账数据，是否覆盖？[确认覆盖] [取消] |

## 四、数据匹配逻辑

### 4.1 匹配优先级

```
Step 1：精确匹配
  身份证号 + 险种 + 费用月份 三者完全一致
  → 匹配为同一条记录，进行金额比对

Step 2：容缺匹配（仅用于差异判定，不自动合并）
  身份证号 + 险种一致，费用月份差1个月
  → 提示："该记录费用月份与系统不一致，是否为同一笔？"

Step 3：未匹配
  台账有系统无 → 差异类型：台账多
  系统有台账无 → 差异类型：系统多
```

### 4.2 金额差异判定

```
差异额 = 系统单位缴费额 + 系统个人缴费额
       - (台账单位缴费额 + 台账个人缴费额)

差异额 = 0         → 完全一致
差异额 > 0         → 系统多（系统金额大于台账）
差异额 < 0         → 台账多（系统金额小于台账）

注意：基数不同但合计金额相同的情况，仍判定为"金额差异"，
因为基数差异代表后续计算逻辑可能存在问题。
```

## 五、客户整理要求说明

### 5.1 发送给客户的要求模板

```
【社保台账整理要求】

尊敬的客户，感谢配合我们的对账复核工作。请按以下要求整理并提供
贵司 [YYYY年MM月] 的社保公积金台账数据：

一、文件要求
   1. 格式：Excel（.xlsx 或 .xls）
   2. 命名规范：公司名称_费用月份_台账.xlsx
      示例：某某科技有限公司_2026-04_台账.xlsx

二、内容要求
   1. 包含当月所有在我司参保员工的缴费明细
   2. 需按险种分列（养老、医疗、失业、工伤、生育、公积金）
   3. 一个员工同一月份有多条记录（不同险种）请分行填写

三、字段要求
   ┌─────────┬────────────────────────────┐
   │ 字段名   │ 说明                       │
   ├─────────┼────────────────────────────┤
   │ 姓名     │ 员工真实姓名                 │
   │ 身份证号 │ 18位纯数字，不含空格         │
   │ 险种     │ 养老保险/医疗保险/失业保险/  │
   │          │ 工伤保险/生育保险/住房公积金  │
   │ 费用月份 │ YYYY-MM格式                │
   │ 缴费基数 │ 当月申报基数，数字格式        │
   │ 单位缴费额│ 当月单位实缴金额            │
   │ 个人缴费额│ 当月个人实缴金额            │
   └─────────┴────────────────────────────┘

四、注意事项
   1. 身份证号中如有"*"号或"X"需要转为大写
   2. 不要包含任何敏感敏感信息（如银行账号等）
   3. 如原始导出格式无法满足要求，请附上原始文件

如有疑问请联系：[联系人] [电话/邮箱]

谢谢配合！
```

### 5.2 常见整理问题及解决方案

| 客户问题 | 解决方案 |
|---------|---------|
| 社保局只能导出PDF，不能导出Excel | 提供PDF，我方协助转换为Excel |
| 一个险种一页Sheet，无法整合 | 单独提供每个Sheet，我方协助合并 |
| 身份证号中间几位是"*" | 联系社保局提供完整数据，或使用实名认证获取 |
| 缴费基数和实缴金额在两个表里 | 提供两个表，我方协助关联 |
| 只能导出当月实时数据，无法指定历史月份 | 尝试补打回单，或使用月度汇总报表 |
