# 统一 Timeline 异动记录展示格式 (Audit Log Standardization)

## 1. 目标
将目前 Timeline 中所有散落的单行修改 (`.change-list`) 或是普通说明文本（如入职、个税报送、转正等），全部强制套入最新的 `.sub-record-card`（嵌套卡片）结构中。
这意味着**每一条异动记录**的 `.timeline-body` 内部，都必须至少包含一个 `.sub-record-card`，并且在卡片头部（`.sub-record-header`）明确标注该动作是“新增”、“编辑”还是“删除”。

## 2. 结构改造规范

针对每一个现有的 `timeline-item`：
1. **清空或保留 `.timeline-note`**: 将原先直接写在 `.timeline-body` 内部的说明文字作为整体操作描述保留在顶部。
2. **插入 `.sub-record-list`**: 将原有的内容（如 `ul.change-list` 或说明文字）包裹在一个 `.sub-record-card` 中。
3. **增加 `.sub-record-header`**:
   - 带有对应颜色的操作 Tag (`tag-green` 为新增, `tag-blue` 为编辑, `tag-red` 为删除)
   - 带有操作对象的文本（如 `基础信息`、`任职信息`、`人员报送信息`、`个税报送信息` 等）。
4. **调整内部边距与底色**: 移除原有的 `.change-list` 上的灰色背景和边框，因为现在外部已经有了 `.sub-record-card` 的边框和白底。

## 3. 具体节点改造映射

- **节点 1: 修改人员档案**
  - Tag: `编辑` (`tag-blue`)
  - 对象: `基础信息`
  - 内容: 原手机号、政治面貌的 `change-list`
- **节点 2: 人员报送**
  - Tag: `新增` (`tag-green`) 或 `编辑` (`tag-blue`) -> 根据业务通常为初次报送或更新，这里统一用 `编辑` 或 `新增`，暂定用 `新增` 标记初次报送。
  - 对象: `人员报送信息`
  - 内容: 原报送字段列表
- **节点 3: 导入报送信息**
  - Tag: `编辑` (`tag-blue`)
  - 对象: `个税专项附加扣除`
  - 内容: 原导入字段列表
- **节点 4: 修改报送信息**
  - Tag: `编辑` (`tag-blue`)
  - 对象: `个税附加扣除`
  - 内容: 原残疾烈属字段列表
- **节点 5: 部门调动**
  - Tag: `编辑` (`tag-blue`)
  - 对象: `任职信息`
  - 内容: 原部门、汇报对象变更列表
- **节点 6: 转正审批通过**
  - Tag: `编辑` (`tag-blue`)
  - 对象: `任职信息`
  - 内容: 员工状态 (试用期 -> 正式)
- **节点 7: 员工入职**
  - Tag: `新增` (`tag-green`)
  - 对象: `员工档案`
  - 内容: 暂无字段列表，保留一句话说明

## 4. 下一步
确认上述改造规范与映射关系后，我将为您执行重构计划，将页面上所有的时间轴节点统一为这种“卡片+状态标识”的展现形式。
