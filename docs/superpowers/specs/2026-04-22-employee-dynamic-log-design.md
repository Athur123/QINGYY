# 员工动态记录Tab设计

## Context

在员工详情页中新增"动态记录"Tab，将现有的右侧边栏动态记录独立展示，同时合并档案变更记录。记录员工因入职、离职、调动、转正、重新入职、编辑、修改人员档案、人员个税报送等生成的档案变更记录。

## 设计方案

### 1. 位置

在员工详情页Tab导航中新增"动态记录"Tab（最后一个位置）。

### 2. 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│  动态记录                                    [时间范围 ▾]   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ● 2026-04-22 14:30  修改了 基本信息                       │
│    操作人: 张三                                             │
│    变更: 手机号码 185****2367 → 186****1234               │
│                                                             │
│  ● 2026-04-21 10:30  转正                                 │
│    操作人: 系统                                             │
│    变更: 试用期状态 试用 → 正式                            │
│                                                             │
│  ● 2026-04-20 09:15  调岗                                 │
│    操作人: 李四                                             │
│    变更: 岗位 Java开发 → 高级Java开发                       │
│                                                             │
│  ● 2026-03-01  参保报送                                   │
│    操作人: 系统                                             │
│    变更: 社保缴纳 深圳市                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3. 时间线样式

- **圆点 + 竖线连接**：左侧时间线，圆点直径10px，竖线1px
- **操作类型标签**：右侧显示操作类型
- **可折叠展开**：点击展开/收起查看变更详情
- **时间范围筛选**：顶部右侧支持按时间范围筛选

### 4. 事件类型

| 类型 | 标签颜色 | 说明 |
|------|----------|------|
| 入职 | success | 新员工入职 |
| 转正 | success | 试用期转正式 |
| 调岗 | primary | 岗位变动 |
| 调薪 | primary | 薪资调整 |
| 离职 | danger | 员工离职 |
| 重新入职 | success | 离职后再入职 |
| 信息变更 | gray | 编辑修改档案 |
| 参保报送 | primary | 社保公积金报送 |
| 个税报送 | primary | 税务申报 |

### 5. 数据结构

```javascript
{
  eventId: "001",
  eventType: "信息变更", // 入职|转正|调岗|调薪|离职|重新入职|信息变更|参保报送|个税报送
  eventTime: "2026-04-22 14:30:00",
  operator: "张三",
  changes: [
    { field: "手机号码", oldValue: "185****2367", newValue: "186****1234" }
  ],
  isExpandable: true // 是否有详情可展开
}
```

### 6. 交互行为

| 交互 | 行为 |
|------|------|
| Tab切换 | 显示动态记录时间线 |
| 点击展开 | 收起/展开显示变更详情 |
| 时间范围筛选 | 仅显示指定时间范围内的记录 |
| 滚动加载 | 记录较多时支持滚动加载 |

## 文件

- `prototype/employee-detail-v2.html` — 修改员工详情页，增加动态记录Tab

## 验证方式

1. 打开员工详情页
2. 点击"动态记录"Tab
3. 验证时间线样式正确显示
4. 点击展开/收起，验证详情显示正确
5. 测试时间范围筛选功能
