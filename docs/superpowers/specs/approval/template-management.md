# 审批模板管理页面设计方案

## 1. 概述

### 1.1 背景

青阳云 HRO 系统需要对接企业微信/钉钉的审批功能。原方案中审批表单在第三方平台设计，新方案改为**青阳云内置审批模板，通过 API 同步到企业微信获取模板 ID**。

### 1.2 设计范围

| 功能 | 说明 | MVP |
|-----|------|-----|
| 模板列表 | 查看所有审批模板 | ✅ |
| 鉴权配置 | 配置企业微信 corpid、secret、agentid | ✅ |
| 同步操作 | 一键同步模板到企业微信获取 template_id | ✅ |
| 表单配置 | 内置只读，不支持自定义 | ✅ |
| 表单预览 | 只读展示表单字段 | ❌ |

## 3. 数据模型

### 3.1 审批模板表 (approval_template)

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | BIGINT | 主键 |
| name | VARCHAR(100) | 模板名称 |
| type | VARCHAR(50) | 模板类型：finance_receivable, finance_payable, attendance, ... |
| platform | VARCHAR(20) | 平台：wecom, dingtalk |
| wecom_template_id | VARCHAR(100) | 企业微信模板ID（同步后获取） |
| form_config | JSON | 表单配置（内置，只读） |
| sync_status | VARCHAR(20) | 同步状态 |
| auto_sync | BOOLEAN | 是否自动同步 |
| last_sync_time | DATETIME | 最后同步时间 |
| last_error | TEXT | 最后同步错误信息 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**同步状态枚举：**
- `unsynced` - 未同步
- `syncing` - 同步中
- `synced` - 已同步
- `failed` - 同步失败

### 3.2 鉴权配置表 (platform_auth)

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | BIGINT | 主键 |
| platform | VARCHAR(20) | 平台：wecom, dingtalk |
| corpid | VARCHAR(100) | 企业ID |
| secret | VARCHAR(500) | 应用Secret（加密存储） |
| agentid | VARCHAR(50) | 应用AgentId |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 4. 功能设计

### 4.1 模板列表

**列表字段：**
- 模板名称
- 模板类型
- 企业微信模板ID
- 同步状态（图标 + 文字）
- 创建时间
- 最后同步时间
- 操作（详情、同步）

**筛选条件：**
- 全部
- 已同步
- 未同步
- 同步失败

**搜索：** 按模板名称搜索

### 4.2 表单配置（内置）

表单配置为预定义字段，只读展示，不支持自定义。

**财务应收单模板字段：**

| 字段名 | 控件类型 | 选项 |
|-------|---------|------|
| 申请人 | 文本 | - |
| 申请金额 | 数字 | - |
| 收款方 | 文本 | - |
| 费用类型 | 单选 | 服务费、工资、其他 |
| 摘要 | 多行文本 | - |

**考勤审批单模板字段：**

| 字段名 | 控件类型 | 选项 |
|-------|---------|------|
| 申请人 | 文本 | - |
| 申请类型 | 单选 | 请假、加班、调休 |
| 开始时间 | 日期 | - |
| 结束时间 | 日期 | - |
| 时长(天) | 数字 | - |
| 原因 | 多行文本 | - |

### 4.3 同步操作

**手动同步：**
- 用户点击模板行的"同步"按钮，或详情页的"重新同步"按钮
- 调用企业微信 API 创建审批模板
- 更新 template_id 和同步状态

**自动同步：**
- 管理员可开启"自动同步"开关
- 开启后，保存模板配置时自动触发同步
- 默认关闭

**同步流程：**
```
用户点击"同步" 或 自动同步触发
           │
           ▼
    调用企业微信API
    POST /cgi-bin/oa/approval/create_template
           │
           ├─ 成功 → 保存 template_id，状态改为 "synced"
           │
           └─ 失败 → 状态改为 "failed"，记录错误信息
```

### 4.4 鉴权配置

**配置字段（基础配置）：**
- 企业ID (corpid)
- 应用Secret
- 应用AgentId

**配置后自动校验连接：**
- 保存时调用 gettoken 接口验证凭证有效性

## 5. 页面交互

### 5.1 列表操作

| 操作 | 行为 |
|-----|------|
| 点击"详情" | 右侧抽屉打开，显示模板详情 |
| 点击"同步" | 弹出确认框，确认后执行同步 |
| 点击"鉴权设置" | 弹窗打开，显示/编辑鉴权配置 |
| 点击"新建模板" | TODO：新建模板功能（本期不做） |

### 5.2 同步状态显示

| 状态 | 图标 | 颜色 |
|-----|------|------|
| 未同步 | ○ | 灰色 |
| 同步中 | ◐ | 蓝色（旋转） |
| 已同步 | ● | 绿色 |
| 同步失败 | ● | 红色 |

### 5.3 错误处理

- 同步失败时，显示红色状态图标
- 鼠标悬停显示错误详情
- 提供"重新同步"按钮

## 6. API 设计

### 6.1 模板相关

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/approval/templates` | GET | 获取模板列表 |
| `/api/approval/templates/{id}` | GET | 获取模板详情 |
| `/api/approval/templates/{id}/sync` | POST | 同步模板到企业微信 |

### 6.2 鉴权相关

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/approval/auth` | GET | 获取鉴权配置 |
| `/api/approval/auth` | PUT | 保存鉴权配置 |
| `/api/approval/auth/verify` | POST | 验证鉴权连接 |
