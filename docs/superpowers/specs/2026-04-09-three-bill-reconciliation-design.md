# 社保对账复核 - 三单分离方案设计

## 1. 概述

### 1.1 背景与问题

HRO系统中，社保费用由系统根据客户提交的增减员名单进行预收，与社保局实际台账数据存在差异。差异来源包括：

- 汇缴：常规月份的申报与实缴差异
- 补缴：员工漏缴后补缴的费用归属问题
- 调基补差：社保基数调整产生的补差费用

**核心问题**：补缴、调基补差等特殊场景下，系统只知道「费用月份」（被补月份），不知道这笔费用在台账中归属于哪个「实缴月份」，导致对账匹配失败。

### 1.2 解决方案

**三单分离模式**——将社保对账拆分为三个独立的账单类型：

| 账单类型 | 说明 | 匹配方式 |
|---------|------|---------|
| 汇缴账单 | 常规月份社保费用 | 按 身份证+险种+费用月份 精确匹配 |
| 补缴账单 | 漏缴补缴费用 | 按 身份证+险种+费用月份（被补月份）匹配 |
| 调基补差账单 | 基数调整产生的补差 | 按 身份证+险种+费用月份 匹配 |

### 1.3 设计目标

1. 三类账单**严格隔离**，互不干扰
2. 每个账单**独立确认**，互不影响
3. 统一仪表盘**总览全局**，便于管理
4. 匹配行视图**直观对比**系统与台账差异

## 3. 功能设计

### 3.1 仪表盘

展示三个账单的整体状态：

- **已匹配数 / 总数**：显示每个账单的匹配进度
- **确认状态**：已确认 / 待确认
- **差异条数**：当前账单类型的差异数量
- **总体预警**：差异超过阈值时显示警告横幅

### 3.2 汇缴对账

**数据范围**：台账中 `记录类型 = "正常"` 的记录

**匹配Key**：`身份证 + 险种 + 费用月份`

**展示内容**：
- 系统记录（姓名、险种、费用月份、金额）
- 台账记录（姓名、险种、费用月份、金额）
- 匹配状态（已匹配 / 系统多 / 台账多）

### 3.3 补缴对账

**数据范围**：台账中 `记录类型 = "补缴"` 的记录

**匹配Key**：`身份证 + 险种 + 费用月份（被补月份）`

**特殊字段**：
- 费用月份（被补月份）：如 2026-01
- 实缴月份：4月实缴，导入时手动选择

**展示内容**：
- 系统补缴单（姓名、险种、被补月份、实缴月份、金额）
- 台账补缴记录（姓名、险种、被补月份、实缴月份、金额）
- 匹配状态

### 3.4 调基补差对账

**数据范围**：台账中 `记录类型 = "调基补差"` 的记录

**匹配Key**：`身份证 + 险种 + 费用月份（产生补差的月份）`

**业务规则**：
- 系统从调基生效月起，按月生成补差费用
- 台账的费用月份 = 产生补差的月份
- 实缴月份 = 导入时手动选择

### 3.5 独立确认机制

每个账单Tab内有独立的「确认账单」按钮：

1. 点击确认 → 弹出确认提示
2. 确认后：
   - 该账单类型数据被**锁定**
   - 显示锁定标识（已确认）
   - 其他账单类型**不受影响**

**锁定效果**：
- 已确认的账单不能修改对账结果
- 「解除锁定」按钮仅限管理员可见

## 4. 数据模型

### 4.1 汇缴记录

| 字段 | 来源 | 说明 |
|-----|------|-----|
| 姓名 | 系统/台账 | 员工姓名 |
| 身份证号 | 系统/台账 | 唯一标识 |
| 险种 | 系统/台账 | 养老/医疗/失业/工伤/生育/公积金 |
| 费用月份 | 系统/台账 | YYYY-MM |
| 金额 | 系统/台账 | 缴费金额 |
| 记录类型 | 台账 | 固定为"正常" |

### 4.2 补缴记录

| 字段 | 来源 | 说明 |
|-----|------|-----|
| 姓名 | 系统/台账 | 员工姓名 |
| 身份证号 | 系统/台账 | 唯一标识 |
| 险种 | 系统/台账 | 养老/医疗/... |
| 费用月份（被补月份） | 系统/台账 | 补缴的是哪个月 |
| 实缴月份 | 台账 | 实际汇款月份（导入时选择） |
| 金额 | 系统/台账 | 补缴金额 |
| 记录类型 | 台账 | 固定为"补缴" |

### 4.3 调基补差记录

| 字段 | 来源 | 说明 |
|-----|------|-----|
| 姓名 | 系统/台账 | 员工姓名 |
| 身份证号 | 系统/台账 | 唯一标识 |
| 险种 | 系统/台账 | 养老/医疗/... |
| 调基生效月份 | 系统 | 基数调整生效的月份 |
| 费用月份（补差月份） | 系统/台账 | 产生补差的月份 |
| 实缴月份 | 台账 | 实际汇款月份 |
| 金额 | 系统/台账 | 补差金额 |
| 记录类型 | 台账 | 固定为"调基补差" |

## 5. 匹配逻辑

### 5.1 匹配流程

```
导入台账 → 识别记录类型 → 按类型分别匹配
                                    ↓
                    ┌────────────────┼────────────────┐
                    ↓                ↓                ↓
               汇缴匹配        补缴匹配         调基匹配
               (正常记录)      (补缴记录)       (调基补差)
                    ↓                ↓                ↓
               差异分析        差异分析          差异分析
```

### 5.2 匹配规则

| 账单类型 | 匹配Key | 严格隔离 | 跨类型处理 |
|---------|---------|---------|-----------|
| 汇缴 | 身份证+险种+费用月份 | 是 | 视为"无法匹配" |
| 补缴 | 身份证+险种+费用月份（被补月份） | 是 | 视为"无法匹配" |
| 调基补差 | 身份证+险种+费用月份 | 是 | 视为"无法匹配" |

### 5.3 差异类型

| 差异类型 | 定义 | 颜色标识 |
|---------|------|---------|
| 已匹配 | 系统与台账均有记录，金额一致 | 绿色 (#DCFCE7) |
| 系统多 | 系统有记录，台账无 | 红色 (#FEE2E2) |
| 台账多 | 台账有记录，系统无 | 黄色 (#FEF3C7) |

## 6. 台账导入

### 6.1 导入要求

- **必填字段**：姓名、身份证号、险种、费用月份、金额、记录类型
- **选填字段**：实缴月份（补缴/调基时必填）
- **类型标识**：台账必须包含「记录类型」字段，值为"正常"/"补缴"/"调基补差"

### 6.2 补缴/调基实缴月份选择

导入补缴或调基补差记录时：

1. 用户选择本次导入的「实缴月份」
2. 系统自动将该月份写入所有导入记录的 `实缴月份` 字段
3. 台账中已有 `实缴月份` 的记录不受影响

## 8. 非功能性要求

| 要求 | 说明 |
|-----|-----|
| 性能 | 单次导入控制在5分钟内，差异比对在3秒内响应 |
| 容量 | 单月单结算主体最大支持10万条记录 |
| 数据保留 | 台账数据保留24个月 |
| 权限 | 仅管理员和HR运营角色可访问 |
| 确认锁定 | 账单确认后仅管理员可解除 |
