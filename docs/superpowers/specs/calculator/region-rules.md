# 地区社保规则配置设计文档

> **版本**: v1.0
> **日期**: 2026-04-10
> **功能**: 按地区配置社保规则，支持条件组合匹配和版本管理

---

## 1. 业务背景

### 1.1 问题描述

同一个地区，不同派遣单位的工伤参保比例可能不同。员工参保时，系统需要根据实际情况判断使用哪个比例：

- 大部分情况：使用参保户头（参保方案）中配置的比例
- 部分地区：要求按实际派遣单位的工伤比例参保
- 特殊情况：需要根据员工类型、所在项目、岗位等条件灵活判断

### 1.2 现有方案的问题

- 当前使用「子户头」（参保户头=参保方案）方案
- 批量Excel操作时，无法明确指定使用哪个派遣单位的工伤比例
- 规则配置不灵活，无法处理复杂场景

---

## 2. 设计目标

1. **规则可配置**：管理员可根据地区、员工类型、项目等条件配置社保规则
2. **条件组合灵活**：支持AND/OR条件组合，适应复杂业务场景
3. **多险种动作**：每个规则可配置多个险种的不同动作
4. **版本管理**：规则修改保留历史版本，支持回滚
5. **冲突检测**：同一员工匹配多条规则时，报错提示管理员明确优先级

---

## 3. 核心概念

### 3.1 地区层级

严格区分省、市、区三级，不继承：

| 层级 | 说明 |
|------|------|
| 省级 | 如：广东省 |
| 市级 | 如：广东省/广州市 |
| 区级 | 如：广东省/广州市/天河区 |

**匹配规则**：严格按员工参保户头绑定的地区查找规则，不向上继承。

### 3.2 规则条件

支持自定义AND/OR条件组合：

| 条件字段 | 说明 | 示例 |
|---------|------|------|
| 员工类型 | 正式员工/派遣员工/实习生/全部 | `员工类型 = 派遣员工` |
| 所属项目 | 具体项目 | `所属项目 in [项目A, 项目B]` |
| 所属客户 | 具体客户 | `所属客户 = 客户X` |
| 岗位 | 具体岗位 | `岗位 = 技术岗` |
| 职级 | 具体职级 | `职级 in [P5, P6]` |

**条件组合逻辑**：
- 同一字段内多选：OR关系（如：员工类型 in [派遣员工, 实习生]）
- 不同字段之间：AND关系
- 支持嵌套括号控制优先级

### 3.3 规则动作

满足条件后执行的动作，支持多险种组合：

| 动作类型 | 说明 |
|---------|------|
| 派遣单位比例 | 使用员工所属派遣单位配置的工伤比例 |
| 指定比例 | 使用规则中配置的具体比例值 |
| 参保户头比例 | 使用参保户头（参保方案）中配置的默认比例 |

**示例**：
```
动作配置：
├── 工伤：派遣单位比例
├── 养老：指定比例 14%
├── 医疗：参保户头比例
└── 失业：参保户头比例
```

### 3.4 版本管理

- 每次修改规则自动生成新版本
- 保留历史版本记录
- 支持查看历史版本
- 支持回滚到历史版本

| 字段 | 类型 | 说明 |
|------|------|------|

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |

## 5. 规则匹配算法

### 5.1 匹配流程

```
输入：员工信息 + 参保户头地区

1. 查找规则列表
   ├── 根据参保户头地区（省/市/区）查找对应规则
   ├── 仅查找「生效」状态的规则
   ├── 仅查找「当前在生效时间内」的规则
   └── 按 priority 升序排列

2. 条件匹配
   FOR EACH 规则 IN 规则列表:
       IF 规则.条件.全部满足(员工信息):
           记录匹配结果

3. 冲突检测
   IF 匹配到的规则数量 > 1:
       IF 存在同险种的动作冲突:
           报错：「规则冲突，请检查优先级配置」
           列出冲突的规则和险种

4. 执行动作
   FOR EACH 匹配的规则:
       执行规则中的所有动作
       （同险种不同规则的动作不允许共存）

5. 产出结果
   RETURN 各险种应使用的比例
```

### 5.2 默认行为

当某地区未配置任何规则时：
- 默认使用参保户头（参保方案）中配置的比例
- 不报错，不提示

## 7. 员工参保时的规则应用

### 7.1 参保流程集成

```
员工参保时：
1. 业务人员选择员工 + 参保户头
         ↓
2. 系统获取：
   - 员工信息（类型、项目、客户、岗位等）
   - 参保户头地区（省/市/区）
         ↓
3. 规则匹配（见第5章算法）
         ↓
4. 无冲突 → 显示参保预览（含规则来源）
         ↓
5. 有冲突 → 弹窗提示冲突，业务人员联系管理员调整
```

### 7.2 参保预览页的规则展示

```
┌─────────────────────────────────────────────────────────────┐
│ 参保确认                                                    │
│ ───────────────────────────────────────────────────────── │

│                                                             │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ 参保规则来源：天和区规则 v3（派遣员工+项目A）           │ │
│ │                                                             │ │
│ │ 险种    │ 比例来源        │ 比例值  │ 金额（元）        │ │
│ │ ────────────────────────────────────────────────────────│ │
│ │ 工伤    │ 派遣单位A比例   │ 0.6%    │ 48.00            │ │
```

## 8. API 设计

### 8.1 规则管理 API

```
GET    /api/region-rules                    # 获取规则列表（支持分页、筛选）
GET    /api/region-rules/:id                # 获取规则详情
POST   /api/region-rules                    # 创建规则
PUT    /api/region-rules/:id                # 更新规则（自动生成新版本）
DELETE /api/region-rules/:id                # 删除规则（仅草稿状态可删除）
POST   /api/region-rules/:id/clone          # 复制规则
POST   /api/region-rules/:id/enable         # 启用规则
POST   /api/region-rules/:id/disable        # 停用规则
GET    /api/region-rules/:id/history        # 获取版本历史
POST   /api/region-rules/:id/rollback       # 回滚到指定版本
```

### 8.2 规则匹配 API

```
POST   /api/region-rules/match              # 匹配规则
请求体：
{
  "employee": {
    "type": "派遣员工",
    "projectId": "P001",
    "customerId": "C001",
    "position": "技术岗",
    "level": "P5"
  },
  "accountId": "ACC001"  // 参保户头ID（用于获取地区）
}

响应体：
{
  "matched": true,
  "ruleId": "R001",
  "ruleName": "广州天河派遣员工特殊规则",
  "ruleVersion": "v3",
  "actions": {
    "工伤": { "type": "dispatch_ratio", "source": "派遣单位A", "value": 0.6 },
    "养老": { "type": "specified_ratio", "value": 14 },
    "医疗": { "type": "account_ratio", "value": 8 }
  },
  "conflicts": []  // 有冲突时返回冲突信息
}
```

- [ ] 规则修改自动生成新版本，保留历史
- [ ] 员工参保时可按规则自动匹配比例
- [ ] 存在冲突时正确提示，不执行
- [ ] 批量导入时可预览规则匹配结果
- [ ] 未配置规则的地区使用参保户头默认比例
