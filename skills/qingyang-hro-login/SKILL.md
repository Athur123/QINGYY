# 青阳云HR SaaS系统登录与维度切换

## 概述

此skill用于自动化登录青阳云HR SaaS系统，在**HRO维度**下查看页面结构。

## 系统维度说明

青阳云HR SaaS系统支持两个维度：

1. **HRO 维度**（默认）：人力资源外包维度，显示外包业务相关功能和数据
2. **EHR 维度**（内部组织）：企业人力资源管理维度，显示内部HR管理功能和数据

**默认行为**：登录后保持在当前维度（通常是HRO），不主动切换维度。只有在用户明确要求切换时才执行切换操作。

维度切换通过右上角管理员菜单中的"切换内部组织"选项实现。

## 使用流程

### 步骤1: 确认登录信息

使用以下默认值，除非用户明确指定了其他值：

```
登录地址: http://192.168.10.168/
租户名称: test
用户名: admin
密码: 123456
```

**默认行为**: 登录后保持在当前维度（通常是HRO），不主动切换维度。

### 步骤2: 导航到登录页面

使用Playwright导航到登录地址：
```javascript
await page.goto('http://192.168.10.168/');
```

**注意**: 如果系统已有有效会话，可能会自动跳转到首页。

### 步骤3: 处理登录

**情况A: 已自动登录**
- 如果页面直接跳转到首页（如`/home/index`），说明已有有效会话
- 跳过登录步骤，直接执行步骤4

**情况B: 需要手动登录**
- 等待登录表单加载
- 填写租户名称、用户名、密码
- 点击登录按钮
- 等待页面跳转

### 步骤4: 获取页面结构（默认）或切换维度（如用户要求）

**默认行为**:
- 登录后保持在当前维度（通常是HRO）
- 直接获取页面结构，不执行维度切换
- 向用户报告当前所在维度

**如果用户明确要求切换到特定维度**:
1. 先检查当前维度（查看控制台日志中的`setUserDimension`）
2. 如果已经在目标维度，跳过切换，直接获取页面结构
3. 如果需要切换：
   - 点击页面右上角的管理员头像/名称
   - 在弹出的菜单中点击"切换内部组织"
   - 等待页面刷新完成

**验证当前维度的方法**:
- 查看控制台日志中的`setUserDimension`输出（EHR或HRO）
- 观察顶部菜单栏：EHR维度会显示"招聘管理"、"行政管理"、"CRM管理"
- 观察左侧菜单和统计数据的变化

### 步骤5: 获取页面结构

使用 `browser_snapshot` 工具获取页面可访问性快照。

如果用户要求截图，使用 `browser_take_screenshot` 工具。

### 步骤6: 返回结果

向用户展示：
- 登录状态（成功/已有会话）
- 当前所在维度（HRO或EHR）
- 页面结构快照（关键元素摘要）
- 截图（如有需要）

## 页面元素识别指南

### 登录页面
- 租户名称输入框：查找带有"租户"或"tenant"相关标识的输入框
- 用户名输入框：查找带有"用户名"或"username"相关标识的输入框
- 密码输入框：查找带有"密码"或"password"相关标识的输入框
- 登录按钮：查找"登录"或"Login"按钮

### 首页关键元素
- 顶部导航栏：核心人事、智能薪酬、假勤管理、招聘管理（EHR）、行政管理（EHR）、审批中心、CRM管理（EHR）、系统设置
- 左侧菜单：工作台、员工管理、员工关系、配置
- 右上角：管理员头像/名称（用于切换维度）
- 统计数据卡片：在职人数、已离职人数等

### 维度切换控件
- 位置：页面右上角管理员菜单
- 菜单项："切换内部组织"

## 错误处理

如果遇到以下情况，向用户报告并提供解决方案：

1. **登录失败**:
   - 检查用户名/密码是否正确
   - 检查租户名称是否正确
   - 检查登录地址是否可访问

2. **页面无法访问**:
   - 确认登录地址是否可访问（http://192.168.10.168/）
   - 检查网络连接

3. **维度切换失败**:
   - 确认系统是否支持维度切换（查看控制台日志中的`dimension`输出）
   - 检查是否在正确的页面（必须在登录后的首页）

4. **元素未找到**:
   - 页面结构可能已更改，使用最新的快照信息定位元素
   - 等待页面完全加载后再操作

## 注意事项

- 登录信息（特别是密码）应谨慎处理，避免在输出中明文显示完整密码
- 系统可能已有有效会话，会自动登录，注意检查当前页面状态
- 维度切换后页面会刷新，需要等待加载完成
- 不同维度显示的菜单和数据可能不同，注意向用户说明当前维度
- HRO和EHR维度的主要区别：
  - HRO：侧重外包业务，顶部菜单6项
  - EHR：侧重内部管理，顶部菜单8项（增加招聘、行政、CRM）

## 技术细节

- 维度信息可在浏览器控制台日志中查看：`dimension [EHR, HRO]`
- 当前维度可通过日志`setUserDimension`确认
- 页面使用Vue.js框架，注意等待组件渲染完成
